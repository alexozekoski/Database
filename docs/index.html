This library simplifies querying, inserting, updating, and reading database results with a fluent API. You can use raw SQL, a chained query builder, or model classes with annotations.

01

Introduction

The project was designed to work across different database engines using the same API shape. The library works with raw SQL, JSON, and annotated Model classes.

Query Builder

Build queries with where, join, limit, and orderBy.

Model

Map tables to Java classes with @Table and @Column.

JSON

Receive results as JsonArray or JsonObject.

02

Connecting to the database

The library includes specific classes for each database. The common flow is: create the connection, call connect(), run operations, and finish with disconnect(). 

// PostgreSQL or MySQL
public class App {
    public static void main(String[] args) {
        PostgreSQL database = new PostgreSQL(
            "localhost", "5432", "user", "password", "mydatabase"
        );

        if (database.connect()) {
            JsonArray tables = database.tables();
            JsonArray columns = database.columns("users");

            System.out.println(tables);
            System.out.println(columns);

            database.disconnect();
        }
    }
}
// SQLite and Firebird use a file path
SQLite sqlite = new SQLite("folder/mydbfile.db");

if (sqlite.connect()) {
    JsonObject result = sqlite.executeAsJsonObject("SELECT * FROM users");
    System.out.println(result);
    sqlite.disconnect();
}
03

Query Builder

For more complex queries, use database.query() or database.query(Model.class). The builder supports filters, joins, grouping, sorting, and pagination. 

Query query = database.query()
    .table("users")
    .select("id", "name", "email")
    .where("active", true)
    .orWhere("email", "like", "%@example.com")
    .orderByDesc("id")
    .limit(10);

JsonArray rows = query.getAsJsonArray();

Useful methods

  • where and orWhere
  • whereIn, whereNotIn, and between
  • join, leftJoin, and rightJoin
  • groupBy, orderBy, limit, and offset

Output

  • getAsJsonArray()
  • getAsJsonObject()
  • count()
  • first() via QueryModel
04

Using Model classes

The most practical way to use the library is through classes that extend Model. Fields must be public and annotated with @Column. 

@Table("users")
public class User extends Model<User> {

    @Column(value = "id", primary = true, serial = true, insert = false, update = false)
    public Long id;

    @Column("name")
    public String name;

    @Column("email")
    public String email;
}
QueryModel<User> query = database.query(User.class);

ModelList<User> users = query
    .where("name", "John")
    .orderByDesc("id")
    .limit(10)
    .get();

for (User item : users) {
    System.out.println(item.toJsonString(true));
}
User first = database.query(User.class)
    .where("email", "john@example.com")
    .first();

if (first != null) {
    System.out.println(first.toJsonString(true));
}
05

CRUD with Model

Model already includes persistence helpers. You can insert, update, delete, save, and refresh records without writing SQL manually. 

User user = new User();
user.setDatabase(database);
user.name = "John";
user.email = "john@example.com";

// inserts when the record is new
user.insert();

// updates the fields enabled for update
user.name = "John Doe";
user.update();

// saves using update or insert automatically
user.save();

// reloads the current data from the database
user.refresh();

// deletes by primary key
user.delete();
06

Global listeners

The project lets you register global listeners per model class. Use Model.addListener to receive insert, update, delete, and select events for every instance of that entity. 

import com.google.gson.JsonObject;
import github.alexozk.database.model.Model;
import github.alexozk.database.model.ModelListenerAdapter;

Model.addListener(User.class, new ModelListenerAdapter<User>() {
    @Override
    public void afterInsert(User model) {
        System.out.println("Inserted: " + model.id);
    }

    @Override
    public void afterUpdate(User model) {
        System.out.println("Updated: " + model.id);
    }

    @Override
    public void getErrors(User model, JsonObject errors, String type) {
        System.out.println(errors);
    }
});
Tip: use ModelListenerAdapter when you only want to override the callbacks you need.
07

Validation

Validation runs from the model annotations and the project's default validation rules. You can also add custom validation with onValidateColumn. 

import github.alexozk.database.model.Column;
import github.alexozk.database.validation.Validator;

@Override
public void onValidateColumn(Column column, Object value, Validator validator) {
    if ("email".equals(column.value()) && value != null && !value.toString().contains("@")) {
        validator.addInvalid(1, "email is invalid");
    }
}
User user = new User();
user.name = "John";
user.email = "john.example.com";

JsonObject errors = user.validate();
if (errors != null) {
    System.out.println(user.validateToString());
}
08

Transactions

Use executeTransaction to group multiple operations in a single unit of work. The database creates a savepoint and commits or rolls back depending on whether the block finishes successfully or fails. 

try {
    database.executeTransaction(db -> {
        User user = new User();
        user.setDatabase(db);
        user.name = "John";
        user.email = "john@example.com";
        user.insert();

        db.query().table("audit_log")
            .set("message", "user created")
            .update();
    }, ex -> System.out.println(ex.getMessage()));
} catch (Exception ex) {
    ex.printStackTrace();
}
Note: the same method is also available on Model as a shortcut to getDatabase().executeTransaction(...).
09

Safe thread access

For shared access across threads, use SynchronizedDatabase. It wraps a normal database instance and exposes the same API with synchronized methods. 

import github.alexozk.database.SynchronizedDatabase;

Database database = new SynchronizedDatabase(
    new PostgreSQL("localhost", "5432", "user", "password", "mydatabase")
);

database.connect();
database.query(User.class)
    .where("active", true)
    .get();
When to use: when the same database object is shared by multiple routines at the same time.
10

Connection pools

The project includes two pool strategies. PoolDatabase keeps a fixed set of pre-created connections, while PoolDynamicDatabase grows and shrinks the pool between a minimum and maximum size based on demand.

PoolDatabase

Use this when you want a fixed number of connections. The pool is created up front and getNextDatabase() returns the next synchronized database in round-robin order.

PoolDynamicDatabase

Use this when the connection count should vary. The pool creates new connections when demand grows, and releases idle connections when it is above the configured minimum. 

// Fixed-size pool
JsonObject config = new JsonObject();
config.addProperty("jdbc", MySQL.JDBC);
config.addProperty("host", "127.0.0.1");
config.addProperty("port", "3306");
config.addProperty("user", "root");
config.addProperty("password", "password");
config.addProperty("database", "mydatabase");

PoolDatabase pool = new PoolDatabase("app-pool", 5, config);
SynchronizedDatabase db = pool.getNextDatabase();

// Dynamic pool with minimum and maximum connections
PoolDynamicDatabase dynamicPool = new PoolDynamicDatabase("app-dynamic", 2, 10, config);
SynchronizedDatabase dynamicDb = dynamicPool.getAndUseNextDatabase();
dynamicPool.releaseDatabase(dynamicDb);
Difference: PoolDatabase is fixed, while PoolDynamicDatabase expands and contracts based on usage.
11

Queries with JsonObject

QueryModel can also build filters from a JsonObject, which is useful when the query comes from an API or a dynamic UI. 

JsonObject where = new JsonObject();
where.addProperty("name", "John");
where.addProperty("email", "john@example.com");

ModelList<User> users = database.query(User.class)
    .where(where)
    .limit(20)
    .get();
Tip: JSON keys must match the annotated column names in the model.
12

Migration and schema

The library also provides utilities to inspect and create schema. This is useful for discovering tables and columns, checking schema changes, and automating model migrations. 

if (database.connect()) {
    JsonArray tables = database.tables();
    JsonArray columns = database.columns("users");

    System.out.println(tables);
    System.out.println(columns);

    Migration.migrate(database, User.class);
}
Table table = database.migrate(User.class);
char result = table.update(true, true, true);

if (result == 'A') {
    System.out.println("Schema changed");
}
User user = new User();
user.setDatabase(database);
user.refresh();
user.refresh("name", "email");
Note: migration uses the model metadata to keep the table definition consistent. If you need a full rebuild, use Migration.fresh(database, ...).
